# Contributing to Axiora

Thanks for your interest in improving Axiora. Contributions of all sizes are welcome.

## How to Contribute

- Open an issue to discuss a bug, idea, or feature request.
- Submit a pull request with clear context and a focused scope.

## Development Setup

Backend:

```
cd backend
npm install
npm start
```

Electron:

```
cd electron
npm install
npm start
```

## Pull Request Guidelines

- Keep PRs focused on one topic.
- Add a brief description of the problem and the solution.
- If behavior changes, explain how to test it.
- Prefer clear, small commits.

## Code Style

- Keep code modular and readable.
- Add comments where logic is complex or non-obvious.
- Avoid introducing secrets in commits.

## Security

If you believe you have found a security issue, please open a private issue or contact the maintainers before publishing details.
